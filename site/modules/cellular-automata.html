Cellular Automata Grid

A grid of cells, each one a small state machine that only ever looks at its eight neighbors. Pick how much information a cell can hold — a single On/Off bit, one of several discrete states, or a full continuous RGB color — then implement nextState(state, neighbors) { ... } and watch the whole grid evolve from your rule alone. Conway's Game of Life, WireWorld, a Cyclic Cellular Automaton, and a Gray-Scott reaction-diffusion system are all one rule apiece — load any of them, stamp in a glider or a Gosper glider gun, or write something nobody's seen before.

Objective

One local rule, one global pattern

Every cell only ever knows its own state and its eight neighbors' states — there's no global logic anywhere. Start with Conway's Life and stamp in a Glider or the Gosper Glider Gun to see motion emerge from nothing but birth/survival counts. Switch the State Model to Discrete and try WireWorld (a 4-state rule used to build real logic circuits out of cells), Brian's Brain (an oscillating 3-state rule with no stable still lifes at all), or Cyclic CA (drag the "Number of states" slider — the classic result for this rule is a field of slowly rotating spirals, and it's the same rule at any state count from 2 to 16). Finish in Continuous RGB, where a cell's state is a full [r, g, b] color instead of a small integer: Reaction-Diffusion runs a real Gray-Scott chemical simulation, and Continuous Life is a tiny taste of the "Lenia" family of smooth, organic automata. Same grid, same playback controls, same eight neighbors — only nextState() and what a "state" means ever change.

Reference

nextState() API

State Model
Chooses what a "state" is: On/Off is a single 0/1 bit, Discrete is an integer 0..N-1 (N set by the slider), Continuous RGB is a full [r, g, b] color, each channel 0-255.
int nextState(int state, int[] neighbors)
The signature for On/Off and Discrete rules. neighbors is the 8 Moore-neighborhood states, in order N, NE, E, SE, S, SW, W, NW.
int[] nextState(int[] state, int[][] neighbors)
The signature for Continuous RGB rules. state and each of the 8 neighbors are [r, g, b] triples; return the cell's next color as a triple too.
count(neighbors, s)
Helper for On/Off and Discrete rules — returns how many of the 8 neighbors are in state s.
avg(neighbors, ch)
Helper for Continuous RGB rules — returns the average of channel ch (0, 1, or 2) across the 8 neighbors.
clamp(v, lo, hi)
Helper available everywhere — clamps v between lo and hi (defaults to 0 and 255) and rounds to an integer.
STATES
Available inside Discrete rules — the current "number of states" slider value, so a rule like Cyclic CA can use (state + 1) % STATES generically.
return value
The state this cell becomes next generation. Every cell in the grid is evaluated against the same old generation, so order never matters.
Run Rule
Recompiles nextState(...) and applies it going forward as the grid steps.
wrap edges
When on, the grid is toroidal — the right edge's neighbor is the left edge, and the bottom's is the top.
palette / brush
On/Off and Discrete show color swatches to paint with; Continuous RGB shows a native color picker so you can paint any [r, g, b] value directly.
Randomize
Re-seeds the board using the active rule's own starting condition — random noise for most rules, a perturbed patch for Reaction-Diffusion.
Famous Patterns
Stamps a known still life, oscillator, spaceship, gun, or circuit onto the center of the grid (clears the board first).
