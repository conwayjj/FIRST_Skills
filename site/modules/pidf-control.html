Overview

PIDF Control

A simulated elevator carriage fights real gravity and friction. Write a Java double calculate(double current, double target) { ... } — called at a fixed 50Hz, exactly like a real periodic control loop — and return the motor output, -1..1. Gain fields you declare above it (kP, kI, kD, kF, an integral accumulator, a previous error) persist between calls the same way instance fields would on a real robot. Change the setpoint or hit "Disturb" and watch the response.

Objective

Converge fast, without overshoot or steady-state sag

Start with P Only and watch it settle noticeably below the target line — gravity is constantly pulling the carriage down, and a proportional term alone can't fully cancel a constant force (it needs some standing error to produce standing output). Add a D term to damp the approach, an I term to slowly erase that remaining steady-state error (with an integral clamp so it can't wind up unboundedly), or skip the wait entirely with a kF feedforward term sized to roughly cancel gravity on its own — the PIDF example combines all four. Change the setpoint mid-flight or hit Disturb to see how each combination recovers.

Reference

calculate(current, target) API

calculate(current, target)
The method you implement; called once every 20ms with the carriage's current position (0-100) and the target setpoint. Return motor output, -1..1.
state fields
Variables declared above calculate() (e.g. double integral = 0;) persist between calls, the same way instance fields would in a real subsystem.
Run / Stop
Compiles calculate() and starts (or stops) the fixed-50Hz simulation loop.
Setpoint
Low/Mid/High presets or the Custom slider — changing it resets the overshoot/settle-time trackers.
Disturb
Gives the carriage an unexpected shove without changing the setpoint — a quick disturbance-rejection test.
Mass, Gravity, Friction
Plant parameters — higher gravity or mass makes steady-state sag (and the need for kF) more obvious.
