Mecanum Drive

Drag the joystick to command forward/strafe translation, drag the rotate slider to spin in place, then write a Java step(dt) that turns those three numbers into four independent wheel speeds with setWheelSpeeds(fl, fr, bl, br). Each wheel is slew-rate limited (it can't jump speed instantly), and the chassis's actual motion is derived purely from your four wheel speeds — get the mixing formula wrong and the robot won't go where the joystick says.

Objective

Mix translation and rotation into four wheels

A mecanum wheel's rollers sit at 45°, so each wheel's speed is a combination of forward/back, strafe left/right, and rotation — the same combination every FRC mecanum robot uses (WPILib's MecanumDrive.driveCartesian): fl = y + x + rot, fr = y − x − rot, bl = y − x + rot, br = y + x − rot. If any wheel's magnitude comes out above 1, scale all four down proportionally so the robot keeps driving in the intended direction, just slower — dropping that normalization step (or a sign) is the most common mecanum bug, and you'll see it immediately as skew or drift instead of clean strafing. The Missing Rotation example shows what happens when the rotation terms are simply left out.
