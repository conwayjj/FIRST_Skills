Filters

Feed a noisy, glitchy, or hand-drawn signal into a small Java method and watch the effect immediately. Implement double filter(double x) { ... return y; } — called once per incoming sample, exactly like a real embedded control loop would call it — and any state you declare above it (a running average, a previous sample, a threshold) persists between calls. Six signal generators are provided, including a Draw It mode so you can sketch pathological inputs by hand.

Objective

Shape the output trace

Start from Moving Average on the Step signal and watch how a 3-tap FIR blurs a sharp edge. Switch to Spiky and try Outlier Reject to see rejection versus smoothing tradeoffs. Then write your own — a filter is just a function of the current sample plus whatever state you choose to remember. The signal generators (including a hand-drawn one) let you stress-test a filter against the same kinds of noise, glitches, and edges a real sensor produces.

Reference

filter(x) API

filter(x)
The method you implement; called once per input sample n, in order. Return the output y[n].
state fields
Variables declared above filter(x) (e.g. double prevY = 0;) persist between calls, the same way instance fields would in a real embedded loop.
Run Filter
Recompiles filter(x) and re-runs it across every sample in the current input signal.
auto-apply
When checked, automatically re-runs your last-compiled filter whenever the input signal changes.
N
Number of samples in the window (Length slider), 16–128.
Draw It
Click/drag on the small canvas to sketch x[n] by hand instead of using a generator.
