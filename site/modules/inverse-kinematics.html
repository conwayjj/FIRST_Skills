Overview

Inverse Kinematics

A vertical linear actuator (prismatic joint) topped with a 1–3 link rotational chain. Click the workspace to place a target, then write a Java step(dt) solver to drive the end effector there. The simulation is inertia-aware — torque limits, damping, and moment-of-inertia all push back on your solver the way a real mechanism would.

Objective

Get the end effector on target

Click anywhere in the workspace to set a target, then hit Start — the default solver uses Cyclic Coordinate Descent (CCD), working from the outermost joint inward, to aim the chain at the target each frame. Try changing the joint count, link lengths, and physics settings on the left, then adjust (or replace) the solver to match. The simulation enforces realistic torque and speed limits, so a naive solver will overshoot or lag — that's the part worth solving for.

Current extension velocity in px/s.

Command rotational joint i's target world-space angle.
getAngle(i)
Current actual angle of rotational joint i.
getVelocity(i)
Current angular velocity of joint i, in °/s.
getInertiaScale(i)
Relative inertia of joint i — higher means more sluggish.
stopSim()
Call once converged to stop the simulation loop.
println(...)
Write a line to the console panel below the editor.
endX, endY
Current end-effector position.
targetX, targetY, error
Target position and current distance to it.
numRotJoints, lengths[]
Configured rotational joint count and link lengths.
